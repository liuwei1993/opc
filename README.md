# OPC Deep Dive：一人公司在AI时代的生存与发展

> 一个人 + AI = 过去需要10人团队才能做到的事。但"能做到" ≠ "能做成"。

## 项目简介

这是一个 **AI驱动的万轮深度知识工程**（Deep Dive Q&A）项目，通过系统化的循环迭代方法，穷尽"一人公司（One-Person Company）在AI时代"这一命题的所有维度。

**核心假设**：借助AI的深度推理能力，一个人完全有能力构建、运营和发展一家有竞争力的公司——但前提是必须系统性地覆盖所有关键决策维度，避免盲区。

## 方法论：10000轮循环迭代Q&A

本项目采用**角度覆盖 + 累积总结**的方法论：

1. **角度覆盖**：从10个大维度、76个子方向系统性地提出问题
2. **循环迭代**：每个回答基于之前所有轮次的累积知识，问题逐轮深化
3. **状态追踪**：通过 checklist 追踪每个子方向的覆盖深度
4. **知识沉淀**：所有洞察持续累积到 `summary.md`

每个Q&A回答要求：
- ≥200字，有案例和数据支撑
- 可执行的步骤而非空泛建议
- 多角度分析利弊
- 针对不同场景给出差异化建议

## 项目结构

```
opc/
├── README.md              # 项目说明（本文件）
├── topic.md               # 核心命题定义与初始问题
├── summary.md             # 累积总结（随迭代持续增长）
├── journal/               # 迭代日志
│   ├── round-001.md       # 第1批：方向选择基础
│   ├── round-002.md       # 第2批：踩坑锦囊
│   ├── ...
│   └── round-356.md       # 第356批：最新产出
└── references/
    └── angle-checklist.md # 角度覆盖清单（状态追踪）
```

### 文件说明

- **`topic.md`** — 项目的起点，定义了核心命题和初始问题
- **`summary.md`** — 知识中枢，所有轮次的洞察在此累积，是后续迭代的上下文基础
- **`journal/round-NNN.md`** — 每批Q&A的完整记录，包含问题、回答和本批总结
- **`references/angle-checklist.md`** — 76个子方向的覆盖状态追踪表（⬜未开始 / 🔄部分覆盖 / ✅已充分覆盖）

## 运行方式

项目采用 **手动会话 + 自动化Cron** 双模式并行推进：

- **手动会话**：大批量深度推理（每批20+轮），适合复杂跨维度问题
- **Cron自动化**：每3小时自动执行30轮，持续推进覆盖深度

自动化运行使用 Qwen 大模型（通过阿里云百炼 DashScope API），确保回答质量。

## 如何使用本项目

### 作为知识库阅读

直接用 Obsidian、VS Code 或任何 Markdown 阅读器打开此仓库：

```bash
git clone https://github.com/liuwei1993/opc.git
cd opc
# 用 Obsidian 打开整个目录作为 Vault
# 或用 VS Code: code .
```

### 快速导航

- 想了解**总体结论** → 读 `summary.md`
- 想了解**特定主题的深度分析** → 在 `journal/` 中按关键词搜索
- 想了解**覆盖进度** → 查看 `references/angle-checklist.md`

本项目的产出是一份**系统化的OPC运营知识图谱**，覆盖从选赛道到心态管理的完整链路。适合：

- 正在考虑或已经启动一人公司的创业者
- 希望用AI工具最大化个人产出的独立开发者
